About This Lab Notebook

This site documents my experience as an undergraduate researcher in the Boston University Robotics Lab. Each week I record what I worked on, challenges I encountered, and what I learned. The Skills section summarizes the technical and professional competencies I've built over the course of my research.

Weekly Research Log

W1

Onboarding & Lab Orientation

January 22 – 26, 2024

First week in the lab! Focused on getting set up and understanding the research direction.

  • Met with my faculty advisor and graduate student mentor to discuss the lab's current projects.
  • Toured the robotics workspace and learned lab safety protocols.
  • Set up my development environment: ROS 2 (Humble), Ubuntu 22.04, and VS Code.
  • Read introductory papers on the lab's focus area in mobile robotics and SLAM.
ROS 2 Onboarding SLAM Ubuntu
W2

ROS 2 Fundamentals & First Publisher Node

January 29 – February 2, 2024

Dove into the ROS 2 framework by working through tutorials and writing my first node.

  • Completed the official ROS 2 Humble beginner tutorials (nodes, topics, services, parameters).
  • Wrote a simple Python publisher node that streams simulated sensor data.
  • Learned how to use ros2 topic echo and rqt_graph for debugging.
  • Attended weekly lab meeting and presented a short summary of SLAM fundamentals.
ROS 2 Python Publisher/Subscriber
W3

Sensor Integration – LiDAR Basics

February 5 – 9, 2024

Started working with the lab's Hokuyo UST-10LX LiDAR sensor in a simulated environment.

  • Studied the sensor_msgs/LaserScan message type and its fields.
  • Visualised LiDAR point clouds in RViz2 using a Gazebo simulation of our robot platform.
  • Wrote a subscriber node to filter out-of-range readings and republish cleaned data.
  • Began reading Thrun et al. Probabilistic Robotics – occupancy grid chapter.
LiDAR Gazebo RViz2 Sensor Fusion
W4

Introduction to SLAM with Nav2

February 12 – 16, 2024

Explored the Nav2 stack and ran SLAM Toolbox for the first time on simulated data.

  • Launched the Nav2 bringup on a TurtleBot3 Gazebo simulation.
  • Used SLAM Toolbox to build and save a 2-D occupancy map of a simulated warehouse.
  • Experimented with different SLAM parameters (linear/angular update distances, loop closure).
  • Documented findings and issues in the lab wiki.
Nav2 SLAM Toolbox Mapping TurtleBot3
W5

Python Data Analysis & Matplotlib Visualisation

February 19 – 23, 2024

Wrote analysis scripts to evaluate SLAM accuracy against ground truth from the simulator.

  • Parsed ROS bag files with rosbag2_py to extract odometry and SLAM pose topics.
  • Calculated Absolute Trajectory Error (ATE) and Relative Pose Error (RPE).
  • Created comparison plots in Matplotlib; identified a drift issue in the SLAM output.
  • Presented results at lab meeting — received feedback to tune the pose graph optimiser.
Python Matplotlib ROS Bags Data Analysis
W6

Hardware Bring-Up on Physical Robot

February 26 – March 1, 2024

Transitioned from simulation to the lab's physical differential-drive robot platform.

  • Configured the Raspberry Pi 4 compute module, motor drivers, and IMU (BNO055).
  • Flashed and calibrated the LiDAR and verified data through ROS 2 on the physical unit.
  • Debugged a timing issue between the motor encoder topics and the odometry node.
  • Successfully drove the robot manually via a joystick teleop node.
Hardware Raspberry Pi IMU Debugging
W7

Autonomous Navigation Test Run

March 4 – 8, 2024

First fully autonomous navigation test in the hallway outside the lab.

  • Pre-mapped the 30-metre hallway using the physical robot + SLAM Toolbox.
  • Sent goal poses through Nav2 and observed the robot navigate around a dynamic obstacle (a cart).
  • Recorded a ROS bag of the run for post-processing; analysed localisation jitter at corridor bends.
  • Filed a lab report summarising observations and tuning recommendations.
Autonomous Navigation Nav2 Localisation Field Testing
W8

Computer Vision – Object Detection Prototype

March 11 – 15, 2024

Started a side-track integrating a USB camera with YOLOv8 for real-time object detection.

  • Installed Ultralytics YOLOv8 and integrated it into a ROS 2 node that publishes detection bounding boxes.
  • Achieved ~15 FPS on the Raspberry Pi 4 by quantising the model to INT8 via ONNX Runtime.
  • Visualised detections overlaid on the robot's camera feed in RViz2.
  • Read the YOLOv8 paper and compared to YOLOv5 baseline performance on the COCO dataset.
Computer Vision YOLOv8 ONNX Camera
